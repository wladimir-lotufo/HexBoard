Descrição

O ContentFull é um wrapper de conteúdo usado em StdView para envolver o conteúdo principal da página. Diferente do Content, ele ocupa toda a largura disponível sem adicionar margens laterais, ideal para grids e tabelas.

Quando Usar

  • Logo após o cabeçalho Razor em um StdView
  • Quando você precisa de toda a largura disponível
  • Para páginas com grids/tabelas grandes
  • Para dashboards com múltiplos painéis
  • Quando o conteúdo precisa ocupar toda a largura da tela

Content vs ContentFull

  • Content - Adiciona margens laterais (padding lateral)
  • ContentFull - Ocupa toda a largura disponível (sem padding lateral)
  • Use Content para páginas normais com formulários
  • Use ContentFull para grids, tabelas e dashboards

Estrutura Hierárquica

  • StdView (cabeçalho Razor)
  • └─ ContentFull (div.content-full)
  •     └─ Row (div.row)
  •         └─ Col (div.col-lg-*)
  •             └─ UIComponents (Panel, Grid, Form, etc.)

Classes CSS

  • .content-full - Classe principal do componente
  • Definida em: Content/style.css

Observações Importantes

  • Sempre deve conter pelo menos um Row dentro
  • Não use Content dentro de ContentFull ou vice-versa
  • O ContentFull é aplicado logo após o cabeçalho Razor do StdView
  • Ideal para maximizar o espaço horizontal disponível

Bibliotecas Necessárias

  • Bootstrap (grid system)
  • style.css (definição da classe .content-full)
